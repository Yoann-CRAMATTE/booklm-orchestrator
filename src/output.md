# Output Module

Takes all instruction blocks from `generator.md` and creates final professional document.

Output format: DOCX or TXT (user preference from questionnaire)

---

## CRITICAL: Language Requirement

**Entire output document MUST be in user's language.**

- **All section headers** → User's language
- **All instructions** → User's language
- **All explanations** → User's language
- **All metadata** → User's language
- Technical terms: Keep original + explain in user's language

Example:
- If user is French: Document titles, instructions, usage guide all in French
- If user is English: All in English
- If user is Russian/Chinese/Spanish: All in that language

---

## Document Structure

Final document contains:

1. **Title Page / Header**
2. **Executive Summary** (analysis overview)
3. **Segmentation Table** (all segments listed)
4. **Instruction Blocks** (one per segment, ready for BookLM)
5. **Usage Guide** (how to use with BookLM)

## Section 5: Usage Guide

**Content - How to use with BookLM:**

```
USAGE GUIDE
===========

How to Generate Your Content with BookLM

Step 1: Go to BookLM
- Visit https://notebooklm.google.com
- Sign in with Google account (or create one)

Step 2: Create Notebook
- Click "Create new notebook"
- Choose your notebook name (e.g., "[Course Name] - Podcasts")

Step 3: Add Your Course
- Click "Add sources"
- Upload the PDF/DOCX/TXT of your course
- Or paste course content directly
- Wait for upload to complete

Step 4: Create Content
- Click "Create"
- Select format:
  - Podcast (if selected format)
  - Slides (if selected format)
  - [other formats based on selection]

Step 5: Use Custom Instructions
- Click "Custom instructions" (or settings icon)
- Copy instruction block from this document (COPY-PASTE section)
- Paste into the Custom Instructions field
- Review the instruction to make sure it's complete

Step 6: Generate
- Click "Generate" or similar button
- Wait for BookLM to create your content (usually 1-5 minutes)
- Listen/view/review the result

Step 7: Review & Iterate (Optional)
- Is the quality good? If yes, download/share
- Not quite right? Adjust custom instructions and try again
- Mix and match segments from different instruction blocks

---

TIPS FOR BEST RESULTS

Do:
- Use complete course material (all pages)
- Copy entire instruction block
- Paste into "Custom Instructions" exactly
- Review BookLM output (listen to podcast, check slides)
- Try one segment first before committing to all

Don't:
- Use partial course material
- Modify instruction block (it's optimized)
- Skip the Custom Instructions field
- Expect perfect output on first try

---

TROUBLESHOOTING

Problem: BookLM says "instruction too long"
Solution: Paste to text editor first, or copy in smaller chunks

Problem: Generated content quality is poor
Solution: Try again (sometimes retry succeeds), or adjust instruction emphasis

Problem: Content doesn't match instruction
Solution: Make custom instruction more specific, include more examples

Problem: Format looks wrong
Solution: Ensure you selected right format (Podcast vs Slides vs Guide)

---

FORMAT EXPECTATIONS

Podcast:
- ~150 words per minute of conversational audio
- Engaging, natural speaking voice
- Includes transitions between topics
- Estimated duration shown in instruction

Slides:
- 1-2 concepts per slide
- Bullet points, not paragraphs
- Placeholder for speaker notes
- Professional formatting

Study Guide:
- Q&A format throughout
- Detailed explanations
- Examples and practice questions
- Textbook-style content

Flashcards:
- Question → Answer pairs
- Memory cues included
- Multiple difficulty levels
- Spaced repetition recommendations

Video:
- Voiceover script (timed to visuals)
- Scene descriptions with B-roll suggestions
- Professional production notes
- Estimated total duration

---

NEXT STEPS AFTER GENERATION

1. Review generated content (listen, read, or quiz yourself)

2. Share or Download:
   - Most platforms allow export (audio, PDF, etc.)
   - Share link with friends/students
   - Download for offline use

3. Iterate if needed:
   - Regenerate with adjusted instruction
   - Mix segments from different runs
   - Combine with other content

4. Provide Feedback:
   - What worked well?
   - What could improve?
   - Help make BookLM-Orchestrator better

---

ADDITIONAL RESOURCES

- Full specification: See SPECIFICATION.md
- Best practices: See guides/BEST_PRACTICES.md
- Troubleshooting: See guides/TROUBLESHOOTING.md
- Getting started: See guides/GETTING_STARTED.md

---

Questions or feedback? Open an issue on GitHub.
```

## Document Formatting (DOCX)

If DOCX format selected:

**Styling:**
- Title: Large, bold, centered
- Section headers: Bold, larger font
- Tables: Professional formatting, alternating row colors
- Code/instructions: Monospace or highlighted background
- Emphasis: Bold for important terms
- Page breaks: Between major sections

**Metadata:**
- Title: "[Course Name] - BookLM-Orchestrator Output"
- Author: "BookLM-Orchestrator Skill"
- Created: [date generated]
- Subject: "Course Segmentation & BookLM Instructions"

If TXT format selected:

**Styling:**
- Plain ASCII text
- Clear section headers with lines (====)
- Indentation for clarity
- ASCII tables where needed
- NO special formatting (all plain text)

## Finalization & Delivery

**Do this:**
1. Assemble all sections in order
2. Check formatting is clean and professional
3. Verify all instruction blocks are complete
4. Test: Can instruction be copy-pasted into BookLM? (at least one)
5. Create final DOCX or TXT file
6. Deliver to user

**Output file naming:**
- DOCX: `[CourseTitle]_BookLM_Output.docx`
- TXT: `[CourseTitle]_BookLM_Output.txt`

## Important Notes for Claude

### Document Quality Standards

**Professional:**
- Clean formatting
- No typos or grammar errors
- Consistent style throughout
- Easy to scan and navigate
- Clear section breaks

**Complete:**
- All segments included
- All instruction blocks complete
- No missing information
- Ready to use immediately

**Actionable:**
- User can copy instruction directly into BookLM
- No additional setup needed
- Instructions are specific and clear

### File Delivery

**DOCX Format:**
- Use python-docx or similar if available
- If not, create structured TXT and guide user to convert
- Or provide as well-formatted Markdown that can be imported

**TXT Format:**
- Plain ASCII, portable
- Works everywhere
- Keep consistent formatting
- Use clear section breaks

**Size Consideration:**
- Very large documents may be slow
- If >100 segments, consider splitting into multiple files
- Each file clearly labeled
